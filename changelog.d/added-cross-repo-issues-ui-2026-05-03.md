- Cross-repo GitHub issues now appear in the All-repos view's existing GH Issues section. Each row carries its repo's folder chip; the "Start" button spawns a session in the issue's own repo (not the active server's REPO_ROOT). Open issues only in v1 — closed ones are filtered out client-side. Folder filter dropdown narrows to a specific repo's issues. Archive button is hidden on cross-repo issue rows since closing a foreign-repo issue requires its own context; switch to that repo to manage its issues. Powered by `/api/issues/all` (5-min per-repo cache).
