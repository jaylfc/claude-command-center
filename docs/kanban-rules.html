Kanban column transition rules

Every trigger that moves a Claude Command Center card between columns. Companion to kanban-rules.md.

Explicit — user clicks or drags
Implicit — a signal under the card changed (hook, poll, git, label, process death)
Some explicit actions also persist a verified / archived flag server-side and may auto-close a GitHub issue — that's a mechanism detail, not a separate trigger type.
Live + icebox tiebreak: if a card has both the icebox label AND a live process, it lands in icebox. The explicit human "park this" signal wins over implicit liveness.

Classifier priority order

First match wins. Cards have no stored column field — this is recomputed on every render (every ~10s) per card. See classifyKanbanColumn at static/index.html:4360.

  1. Fresh-session sticky — for ~60s after spawn, pin to first column. Releases on real progress. (Less critical now that planning is gone — kept as a safety net.)
  2. Manual override from drag-drop (localStorage) — unless stale (server flag set, or stage advanced past it).
  3. verified flag → verified
  4. archived flag → archived
  5. Backlog cards: issue CLOSED+COMPLETED → verified · CLOSED other → archived · label needs-attention → needs-attention · label icebox → icebox · else → backlog
  6. label icebox (on issue or linked session) → icebox · wins over liveness — explicit park beats implicit live process
  7. Live + sidecar: any → working (waiting / has_writes / pre-tool — all working)
  8. stage = pushed/committed → review
  9. !live + edits + last=assistant → review (dormant w/ unsaved edits)
  10. label needs-attention → needs-attention
  11. label claude-in-progress → working
  12. Live + pkood (any state) → working
  13. Live (any other) → working
  14. Fallback → working · render adds the blue Idle pill if isIdleSession()

Aliases that confuse

waiting — sidecar status (between Claude turns), not a column. Routes to working.

blocked — internal "needs your attention" classification kind, not a column.

icebox — GitHub label name matches the column name. Adding the label routes the card to the column. Removing it sends the card back to its natural column (typically working if a session is live, otherwise backlog).

claude-in-progress — GitHub label that routes a dead session into working. Says "this is the active branch."

testing — manual-only column. Nothing routes in or out automatically.

verified vs archived — both close the linked GitHub issue, but with different reasons (completed vs not planned).
